# Algraf v0.61.0 Plan

Status: Implemented
Target version: 0.61.0
Owner: Algraf maintainers
Related spec: [`ALGRAF_SPEC.md`](ALGRAF_SPEC.md)
Predecessor plan: [`V0_60_PLAN.md`](V0_60_PLAN.md)
Roadmap theme: Story-chart expressiveness for the Datafarm Studio bike-share case study.

## Purpose

Algraf v0.61.0 adds the small chart-authoring surface needed for the Datafarm
Studio bike-share story to render its intended visual argument directly.

The release is focused on chart-facing expression, not data preparation. PDL
continues to own filtering, joining, aggregation, reshaping, and story-table
materialization. Algraf v0.61.0 should make the prepared tables easy to render
as the story's intended shapes: stacked area, scatter, slope chart, grouped
bars, and ranked horizontal bars.

The motivating acceptance target is the bike-share case study in
`../../datafarm-bikeshare/STORY.md`, using the five prepared CSV outputs
generated by the PDL v0.23 story-table plan.

## Scope

### Stacked Area Layout

Status: Implemented.

`Area` supports a `layout` property accepting `"identity"`, `"stack"`, and
`"fill"`. Existing area behavior remains `"identity"`. For stacked layouts,
rows are grouped by explicit `group` when present, otherwise by categorical
`fill` or categorical `stroke`.

Acceptance criteria:

- `Area(fill: rider_type, layout: "stack", baseline: 0)` renders one polygon per
  rider type over the stacked y ranges.
- Positive and negative values stack separately around the baseline.
- `layout: "fill"` normalizes each x stack to 100 percent.
- Scale-domain training accounts for stacked lower and upper bounds.
- Analyzer diagnostics cover unsupported layout values, non-numeric y values,
  and grouped layouts without a usable grouping aesthetic.
- SVG, draw-list, and raster-equivalence tests cover identity, stack, and fill
  area layouts.

### Categorical Position Domain Ordering

Status: Implemented.

Position-axis scales accept explicit string-array domains for categorical and
nested-band axes:

```ag
Scale(axis: x, domain: ["Share of rides", "Share of revenue"])
```

Acceptance criteria:

- Declared categories lead the trained domain in source order.
- Data categories not listed in the domain are appended in first-appearance
  order with a warning.
- Declared categories with no rows still reserve a band.
- Diagnostics cover duplicate domain categories, empty domains, and string-array
  domains on continuous axes.
- Metadata sidecars report the trained categorical domain after applying the
  explicit order.

### Numeric Formatting for Text

Status: Implemented.

`Text` supports a `format` property for numeric labels:

```ag
Text(label: rev_per_dock, format: "$.2f", dx: 6, anchor: "start")
```

Acceptance criteria:

- Initial supported formats are `.0f`, `.1f`, `.2f`, `$.2f`, `.0%`, `.1%`, and
  `.2%`.
- Formatting is deterministic and locale-independent.
- `timeFormat` continues to handle temporal labels; using both `format` and
  `timeFormat` on the same label is diagnosed.
- Diagnostics cover unknown format strings and numeric formats on non-numeric
  label columns.
- SVG and draw-list output agree on formatted text strings.

### Terminal Label Geometry

Status: Implemented.

Add a `Label` geometry for direct line-end and series-end labels:

```ag
Space(metric * share) {
    Line(group: rider_type, stroke: rider_type)
    Label(label: rider_type, at: "end", group: rider_type, dx: 8, fill: rider_type)
}
```

Acceptance criteria:

- `Label` renders one text mark per group at the start or end row in physical
  x-axis order.
- `at` accepts `"start"` and `"end"` and defaults to `"end"`.
- If `group` is absent and `label` maps to a column, rows are grouped by that
  label column.
- `fill`, `alpha`, `size`, `anchor`, `dx`, `dy`, and `format` follow existing
  text-style behavior where applicable.
- Diagnostics cover invalid `at` values, missing x axes, and numeric formatting
  on non-numeric labels.

### Spec, Editor, and Example Hygiene

Status: Implemented.

The new chart surface must be documented and discoverable before Studio depends
on it.

Acceptance criteria:

- `docs/ALGRAF_SPEC.md` records v0.61.0 behavior for all new properties and
  geometry.
- Registry metadata, completions, hover, signature help, diagnostics, TextMate
  grammar where needed, and formatter support are updated.
- Bike-share story chart examples render without fallback syntax or helper
  label tables.
- Workspace, README, examples, generated graphics, and version stamps are
  aligned before marking this plan implemented.

## Non-Goals

- No PDL-style data preparation in Algraf.
- No `Annotate(...)` geometry for this story; literal-positioned `Text(...)`
  already covers static annotations.
- No `Flip()` chart transform; horizontal bars are expressed by putting the
  numeric measure on x and category on y.
- No `Bar(position: "dodge")`; grouped bars continue to use nested algebra.
- No broad renderer rewrite, new backend, source connector, plugin system, or
  package publication work.
